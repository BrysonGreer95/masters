## 1. Masters Fantasy Scoring — Manual Entry Going Forward

**Problem:** The current system auto-calculates Masters fantasy points by pulling live leaderboard data via the RapidAPI golf API and ranking picks by cumulative score. The real Masters, however, awards points based on birdies, eagles, etc. — not just total strokes. This caused a scoring discrepancy: Marty and Zach tied in the final standings, and Zach only won the tiebreak via his putt guess.

**Decision:** Remove the auto-calculation of Masters fantasy points. Points will be entered manually after the tournament concludes.

**Options for implementation:**
- Keep the existing `fantasy_points` block in `data.json` and just set the values manually each year after the tournament.
- OR add a `masters_results` array in `data.json` where you list the finish order manually, and the app calculates points from the `FANTASY_POINTS` table. Example:
  ```json
  "masters_results": [12, 3, 7, 1, ...]  // player IDs in finish order
  ```
- Either way, the `applyFantasyPoints` mutation in the store and the auto-trigger logic in `Masters.vue` can be removed or disabled.

**Tie situation this year:** Marty tied Zach on points. Zach won via putt tiebreaker. No action needed — just documenting for record.

## 7. Live Score Entry — Players Input Scores on Their Phones

**Goal:** Players enter their own hole scores from the course on their phones, and those scores show up live on the website. Willing to pay for a 3rd party service if it has a proper integration path.

---

### Option A: Golf Genius (Recommended 3rd Party)

**What it is:** Golf Genius is the most widely used tournament management platform in amateur/club golf. It has a dedicated mobile app (iOS + Android) where players tap in their score after each hole.

**How it works for us:**
- Organizer sets up the tournament in Golf Genius (teams, format, tee times)
- Players download the free Golf Genius app, sign in via a tournament code
- Scores are entered hole-by-hole on their phone and sync to Golf Genius's servers in real-time
- Golf Genius has a **REST API (v2)** — you request an API key through their support team
- Our Vue site can poll or subscribe to that API to pull live scores and display them natively

**Pros:**
- Players already know what a golf scorecard looks like — zero learning curve
- Live leaderboard inside the app comes free
- API is documented and used by real clubs and tour associations
- Handles scramble formats natively

**Cons:**
- Pricing is not public — must contact Golf Genius sales; likely $X/year subscription
- API key requires going through their support team, not self-serve
- We'd be dependent on their uptime / format constraints
- Some setup overhead per tournament

**Links:**
- [Golf Genius Tournament Management](https://www.golfgenius.com/tm)
- [Golf Genius API v2 Docs](https://www.golfgenius.com/api/v2/docs)
- [Live Scoring Setup Guide](https://docs.golfgenius.com/article/show/12366-live-scoring-mobile-scoring)

### Option B: Custom Build (PWA + Supabase) — Best Long-Term Fit

**What it is:** Build a simple mobile-friendly score entry page directly into this site (or as a companion PWA), backed by a real-time database. Players open a link on their phone, log in, and tap in scores. The main leaderboard page updates live.

**How it works:**
- Add [Supabase](https://supabase.com) as a backend — free tier is generous, Postgres-based, has real-time websocket support
- Each player gets a login link (or simple PIN) tied to their player ID
- Score entry UI: a simple hole-by-hole card, one input per hole, big buttons for phone use
- On submit, score writes to Supabase → Vue site listens via Supabase Realtime → leaderboard updates instantly with no page refresh
- No app store, no download — just a link they open in Safari/Chrome

**Pros:**
- Full control over data format, UI, and scoring rules
- Integrates natively with our existing Vue + Vuex setup — scores go straight into our data model
- Supabase free tier covers this easily (small group, low traffic)
- No dependency on a 3rd party's tournament format or API availability
- One-time build, reusable every year

**Cons:**
- We have to build and maintain it
- Need to handle auth (even lightweight PIN-based auth) so players can't overwrite each other
- More up-front dev work than Golf Genius

**Rough implementation plan:**
1. Add Supabase project, define `scores` table: `(player_id, hole_index, score, event_type, updated_at)`
2. Replace or supplement `localStorage` persistence in `store/index.js` with Supabase reads/writes
3. Build a `/enter` route in Vue — mobile-optimized score entry form
4. Use Supabase Realtime channel to push score updates to all connected clients (the leaderboard page)
5. Admin still has override capability via `AdminView.vue`

### Option D: Squabbit (Strong Contender)

[Squabbit](https://squabbitgolf.com) is a free golf tournament and league management platform with native iOS and Android apps.

**What it offers:**
- Native iOS & Android apps + web version (no download required for web)
- Apple Watch and Wear OS support
- 30+ scoring formats including scramble
- Live leaderboards
- World Handicap System integration
- Self-registration with Stripe payment integration
- Supports up to 1,000 players and 500 teams
- **Completely free** — no hidden costs, no player/team limits

**Integration plan (until API is available):**
- Email sent to Squabbit requesting a public API — awaiting response
- In the meantime: **nightly CSV import**
  - Export scores from Squabbit at end of each day
  - Build a CSV import tool in AdminView that parses the file and updates `data.json` / store state
  - If Squabbit responds with an API, swap the CSV step for a direct fetch

**Verdict:** Squabbit is the pick. Free, native app, great scoring experience. CSV import is a fine interim solution.
